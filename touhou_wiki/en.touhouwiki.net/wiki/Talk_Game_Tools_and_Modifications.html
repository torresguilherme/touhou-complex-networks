New tool template vs. separate tool pages[edit]

I've created a new template for the tools on this page. This will display the information in a more organized way and eliminate the need for seperate pages for the individual tools and the game tool pages (like Fairy Wars/Tools) in my opinion. And while it's clearly better than the current mess, I'm seeing a few problems with this:

  • The most important tools are pretty self-explanatory or have at least an English readme file included. But there are still a few Japanese ones like the Vertical Play tool where there is no way around a lengthy usage description. A dedicated page would probably still be better in those cases.
  • The support list may be structured a bit awkwardly. Especially on non-widescreen monitors and with big rippers like Brightmoon, it really stretches the page. And I don't think splitting the main series into multiple rows based on generations would be a good idea...
  • Initially, I thought about using the game's Explorer icons (as in, for EoSD) on the support list, instead of the downscaled game covers. But they don't look too bad...

If you have suggestions to improve anything, feel free to edit the template and all connected pages. --NamelessLegacy 01:13, 21 January 2011 (UTC)

Disabling sprites?[edit]

Does anyone know if there's a tool that will disable all sprites in a game? This is so we can printscreen a clear image of the background of a stage without any sprites in the way. ï½ž Tony64 (Talk/Con.) 01:48, 9 August 2015 (UTC)

I'm pretty sure that you can use the Touhou Toolkit to extract the actual image used in the game from the .dat file. --DTM (talk) 03:02, 9 August 2015 (UTC)
Ah, I've already tried that. The actual images are various textures that are used to make the backgrounds, thus not being the 'actual' backgrounds. The textures are mostly 2D images, whilst in-game backgrounds are 3D. The games themselves will turn them into the backgrounds that we see in-game. Unless there is something I'm missing from that tool, that won't work. That's why disabling all sprites would be a better option to printscreen the perfect background image. ï½ž Tony64 (Talk/Con.) 13:21, 9 August 2015 (UTC)
Ah, found a solution, but it's time consuming. What I can do is extract the .dat files with toolkit, extract the .anm files that contain the images I want to remove, make those images 100% transparent, add them back to the .anm archives, add those back to the .dat archive, and now I can printscreen the backgrounds. ï½ž Tony64 (Talk/Con.) 18:06, 9 August 2015 (UTC)
